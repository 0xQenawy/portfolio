Product Owner & Developer

SecurePass Tools

A privacy-first password security toolkit with regional (Arabic) context awareness — catching patterns that mainstream checkers miss entirely.

View Full Breakdown
Problem

Most password-strength checkers ignore regional context — they don't flag common Arabic names/patterns (e.g. "Ahmed", "Mohamed") as weak, which is a real security risk in MENA markets. Many tools also compromise privacy by sending passwords to a server for analysis.

Solution

Leveraged Cybersecurity fundamentals and Secure Product Design to build a privacy-first, fully client-side password toolkit with region-aware pattern detection plus a memorable passphrase generator.

Product Decision

Chose a zero-server, zero-knowledge architecture — trust by design instead of by policy — even though it meant building more detection logic client-side instead of relying on a backend.

Role

Product Owner & Developer — defined the Product Strategy, researched regional password patterns, and led the Software Architecture for the client-side implementation.

Outcome

Live, functional product — directly usable in the browser with no installation required.
